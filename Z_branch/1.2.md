## 项目总体说明与运行方式

- **简介**: 本项目为完成一个英文到中文翻译器的作业（基于 PyTorch），需要补全并运行现有代码以训练/评估翻译模型。
- **关键文件**: `Assignment en2cn_Translator/translator_en2cn.py`（程序入口）、`Assignment en2cn_Translator/Tutorial.ipynb`（训练/评估示例与说明）、`model/transformer.py`（模型实现位置）。
- **数据位置与格式**: 数据存放于 `Assignment en2cn_Translator/data/en-cn/`，包含 `all_data.txt`, `train.txt`, `dev.txt`, `test.txt` 及对应的 `_mini` 版本，建议先用 `*_mini` 进行快速调试。
- **运行环境**: 推荐使用 Miniconda，主要依赖版本如下：Python 3.12，PyTorch 2.5.1（按机器的 CUDA 版本安装对应的 torch），nltk 3.9.1，numpy 1.26.3，matplotlib 3.10.0。
- **运行命令**: 在项目根目录下运行 `python translator_en2cn.py` 启动训练或评估；更多运行细节请参阅 `Assignment en2cn_Translator/Tutorial.ipynb`。
- **建议**: 初次运行先在 `train_mini.txt` / `dev_mini.txt` 上跑通一遍，确认依赖与数据路径正确后再在完整数据上训练；若使用 GPU，请按 README 安装匹配的 `torch + cuda` 版本。
 
## Tutorial.ipynb — 数据流与训练/评估流程要点

- **总体流程**: 从数据预处理 → 构建词表/编码器 → 准备批次与掩码(mask) → 构建 Transformer 模型 → 训练迭代（前向、计算 loss、反向、更新）→ 定期验证与预测测试。
- **数据与预处理**: 使用 `Assignment en2cn_Translator/data/en-cn/` 中的分句文件，笔记本中用 NLTK 分词、统计词频并构建英文/中文词表，输出索引序列等数据预处理步骤。建议先用 `*_mini` 数据做快速调试。
- **构建词表/编码器**: 核心文件为 `tokenization.py` 中的 `PrepareData` 类。流程包括：(1) `load_data` 读取平行语料并分词（英文用 NLTK，中文按字符），添加 BOS/EOS 标记；(2) `build_dict` 用 Counter 统计词频，构建 word_dict（词→ID）、index_dict（ID→词）及 total_words，特殊词 PAD=0/UNK=1；(3) `wordToID` 将词序列转为 ID 序列（需自行实现）；(4) `splitBatch` 分批次并填充对齐，返回 MaskBatch。验证集 batch_size=1，词表仅基于训练集构建。
- **批次与掩码**: 训练数据按批次打包为 `MaskBatch`，包含 `src`, `tgt` 以及 `src_mask`, `tgt_mask`（掩盖 padding 与 decoder 的未来位置）。
验证时要求 batch size=1 以逐例解码比较。
- **模型构建**: 提供 `InputEmbeddings`、`PositionalEncoding`、多层 `Encoder`/`Decoder`、`ProjectionLayer`，以及 `build_transformer()` 函数来组装并初始化模型参数（Xavier 初始化）。
- **训练细节**: 损失使用 `CrossEntropyLoss(ignore_index=PAD)`，可选 label smoothing（示例 eps=0.1）。优化器推荐 Adam（paper 建议 beta1=0.9, beta2=0.98），并配合 warmup 学习率调度：

	$$lrate = d_{model}^{-0.5} \cdot \min(step^{-0.5},\; step\cdot warmup\_steps^{-1.5})$$

- **训练循环**: 每个 batch 执行 `model.encode` → `model.decode` → `model.project`，计算 loss，`loss.backward()`，`optimizer.step()`，`optimizer.zero_grad()`，并维护 `global_step`。每 N 个 epoch（示例为每 5 个）调用 `run_validation()`。
- **验证与预测**: 验证使用 `greedy_decode()`（贪心解码）逐步生成词 ID，转换为中文符号并打印 `SOURCE/TARGET/PREDICTED`。也建议实现 BLEU 评分函数用于量化评估。
- **建议与注意事项**: 首次运行请（1）确认 `PAD/BOS/EOS` id 与词表一致；（2）在无 GPU 时将 `device` 设置为 CPU；（3）先用小数据快速跑通再进行长时间训练；（4）若要改进译文质量，可实现 beam search、调整 label smoothing 或 warmup 步数。

## 数据格式 — `data/en-cn/` 目录文件说明

**文件列表及规模**

| 文件名 | 行数（句对数） | 用途 |
|--------|---------------|------|
| `train.txt` | 14,533 | 训练集（完整） |
| `dev.txt` | 1,817 | 验证集（完整） |
| `test.txt` | 1,817 | 测试集（完整） |
| `train_mini.txt` | 1,000 | 训练集（小样本，快速调试） |
| `dev_mini.txt` | 200 | 验证集（小样本，快速调试） |
| `test_mini.txt` | - | 测试集（小样本） |
| `all_data.txt` | - | 全部数据合并 |

**数据格式**

所有文件均采用 **TSV（制表符分隔）** 格式，每行包含一句英文和对应的中文翻译，以制表符 `\t` 分隔：

```
英文句子\t中文句子
```

**示例**：
```
Anyone can do that.	任何人都可以做到。
How about another piece of cake?	要不要再来一块蛋糕？
She married him.	她嫁给了他。
```

**数据特点**

- **英文**：完整句子，包含标点符号，首字母大写
- **中文**：完整句子，包含标点符号
- **平行语料**：每行是一对一的中英翻译
- **预处理**：英文使用 `word_tokenize` 分词，中文按字符级分词，自动添加 BOS/EOS 标记

**使用建议**

- **快速调试**：先用 `train_mini.txt`（1000句）+ `dev_mini.txt`（200句）跑通流程
- **完整训练**：使用 `train.txt`（约1.4万句）进行正式训练
- **评估**：使用 `dev.txt` 或 `test.txt` 进行模型评估
